Coisinhas importantes do CRM
Autor: Mauro Laranjeira
Data: 17/03/2011 07:36

Este post é dedicado inteiramente ao meu amigo Zé (José Marcelo).
Creio que pelo menos uma vez por semana eu chamo o Zé e pergunto a mesma coisa que havia perguntado semana passada, isso acontece com ele também.
“Qual a Tabela…?”
“Qual transação mesmo…..?”
Resolvi fazer um post com as transações, visões, objetos e classes mais utilizados no CRM, isto pelo menos vai ajudar o Zé e eu a não perguntar sempre as mesas coisas (vou perguntar so mais uma vez para escrever este post, rs :roll:).
Cluster Visões (Transação SM34)
BSPWDVC_CMP_EXT – Cluster com ampliações de componentes;
Criação de Campo no CRM 7.0 com AET – Vídeo
Classes e Interface (SE24)
CL_BSP_WD_MESSAGE_SERVICE –
Mensagem com BOL
CL_CRM_IC_5X_EXT_ALERT_SRV –
Alert Modeler
CL_CRM_BOL_ENTITY – Classe de objeto BOL;
IF_BOL_ENTITY_COL – Interface para trabalhar com collections, tanto BOL como de Nó de Valor;
CL_CRM_BOL_CORE – Core do BOL, pode usar o método get_instance para pegar a instância;
Transações
GENIL_MODEL_BROWSER – Hierarquia de todos os objetos BOL;
GENIL_BOL_BROWSER – Transação para teste de Query BOL (Objetos de pesquisa), com essa transação é possível após a utilização; da query ver os relacionamentos dos objetos atrelados;
BSPWD_CMP_CMPWB – Ampliação, criação é alteração de componentes;
AXTSHOW – Para o CRM 7.0, que a criação de EEWB ocorre na aplicação Web, pode acontecer do SAP “travar” durante a criação de algum campo, está transação faz a correção dos campos / atributos com erros;
PPOMA_CRM  – Estrutura organizacional do CRM, semelhante á PPOME do R3;
SCC1 – Transporte de request entre Client;
WUI_SSO – Abri o WebClient já logado
Tabelas
CRMC_IU_MD_PROF – Configuração de “Confirmação Automática” para o Interaction Center;
Query BOL
BTQuery1O – Consulta de documentos BTOrder;
BuilHeaderSearch – Consulta de parceiro, objeto BuilHeader;
SM30
BSPWDV_EHSET_ASG – Ativa/Desativa o SET de ampliação para o client;
Função
ITS_IMPORT_CONTEXT – Valida de um processo esta sendo executado por transaction launcher
Pretendo criar outros posts explicando algumas transações e algumas configurações, irei fazer referencia neste post 😀
